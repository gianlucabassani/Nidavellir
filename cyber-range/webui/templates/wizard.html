Software-Under-Test Wizard

Bring your own GitHub project up as a software-under-test arena — target, then the setup consent, then review the topology before you launch. For adversarial attack scenarios, use Launch.

What are you standing up?

A fresh Ubuntu box with this repo cloned read-write into /opt/sut; the service is brought up in the setup phase (next step). An optional Kali foothold is added for the engagement.

How it works
Clone — your repo is cloned into /opt/sut on a fresh Ubuntu box.
Isolate — the box sits on its own segment, network-isolated from the host.
Attack — optionally a Kali foothold joins the segment for the engagement.

Setup phase — your consent

Bringing the service up needs a configurator session on the victim. This is the load-bearing safety choice: nothing writes/configures the box without it, and it's time-boxed, step-budgeted, and its egress is revoked before the engagement.

Safety guarantees
Time-boxed — the setup session auto-expires after the configured timeout.
Step-budgeted — hard cap on the number of commands that can run.
Egress revoked — network egress is cut before the engagement begins.
