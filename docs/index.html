Current Functionality

Nidavellir ships with a production-grade substrate decoupled across three core system layers.

Decoupled Substrate

Independent control plane using FastAPI orchestrators, Redis database buffers, and Celery background task runners for clean concurrent actions.

Multi-Cloud Providers

Abstract drivers support local container ranges (docker-local running inside seconds), OpenStack VMs, and AWS instances on demand.

Secrets & Safety

Fernet encryption at rest for target data. Strict network egress isolation, active rate limiting, and log redactions keep host servers safe.

Telemetry & Audits

State machine logs compile everything to an append-only SQL database audit table. Operator consoles review historical action traces.

TTL Stuck Reaper

Automated cleanup engine tracking execution time. Identifies and terminates running ranges once the Time To Live expires to prevent cost leaks.

Community Importers

Automated registry path converts public Vulhub compose environments into validated v3 scenario yaml packages on the fly.

Three Architectural Pillars

Nidavellir is engineered around modularity, security containment, and deep telemetry integration.

01 // Topologies

Dynamic Topologies

Author scenarios inside data-defined YAML templates. Topologies compile on-demand to Docker local engines, OpenStack VMs, or AWS instances using dynamic provider drivers. Supports arbitrary N-node segment setups.

02 // Gateways

MCP Agent Gateways

Secure and regulate Model Context Protocol (MCP) clients. Every action runs through gateway sandboxes enforcing strict network scopes, guardrail controls, budgets, and logging to append-only trace auditories.

03 // Compiler

Zero-to-Prompt Compile

Translate plain text descriptions into operational YAML topology schemas using large language models. Validates generated specifications against strict schemas before exposing infrastructure.

System Architecture

Nidavellir coordinates components to maintain isolated ranges and expose them safely to AI agents.

Modular Decoupled Subsystems

The Flask web console communicates with the FastAPI orchestrator via REST API calls. Tasks are queued inside Redis and dispatched to Celery workers, which handle Docker container builds, AWS AMI deployment, or OpenStack VM configuration. External AI agents connect exclusively through the gateway system which isolates network requests.

Development Roadmap

Follow our sequenced plan detailing completed milestones and planned targets.

Phase 0 Completed

Repositioning & Hygiene

Renamed core user roles (student/instructor to operator/agent). Dropped obsolete EDU-ITS server bindings. Audited vulnerabilities, verified Secrets hygiene log redactions, and established green pytest/ruff gates inside CI pipelines.

Phase 1 Active

Dynamic Topologies (Pillar 1)

Introduce the abstract v3 YAML scenario schema modeling arbitrary networks, node structures, and subnets. Build compiler engines that deploy these dynamic scopes locally to Docker, or VM providers (OpenStack/AWS). Setup the Vulhub community environment importer.

Phase 2 Scheduled

MCP Agent Gateways (Pillar 2)

Deploy secure Model Context Protocol gateway layers. Enforce per-session credentials, stance capabilities (attacker foothold commands, MITM packet sniffers, blue-team defenders), step budgets, and audit tracer records.

Phase 3 Planned

Zero-to-Prompt Scenario Gen (Pillar 3)

Deploy natural language LLM prompt compilers that generate validated v3 yaml topology configurations dynamically. Wire an approval wizard inside the operator dashboard before provisioning code to hypervisors.

Phase 4 Planned

SUT Monitoring & Scoring Benchmarks

Integrate software-under-test sidecars on target machines to catch runtime crashes, sanitizers, or exceptions. Establish dynamic agent scoring and benchmark trackers evaluating model efficiencies against open-ended vulnerabilities.
